/ˈruːbi/

The name "Ruby" originated during an online chat session between Matsumoto and Keiju Ishitsuka on February 24, 1993, before any code had been written for the language.

http://en.wikipedia.org/wiki/Ruby_(programming_language)

Ruby is

  • balance between functional programming and imperative programming
  • focusing on making programming more natural
  • fully object-oriented
  • easy to understand

History

First public release of Ruby 0.95 in japanese on December 1995.

First official English mailing list in 1993, following the Ruby 1.3 release

Ruby 1.8 released in August 2003, the most used and most stable release (retired in 2013)



In 2005, the first release of Ruby on Rails framework. RoR is often credited with making the Ruby famous.

Ruby 1.9 introduced in December 2007 with a lot of backward incompatible changes and features.

Philosophy

Often people, especially computer engineers, focus on the machines. They think, "By doing this, the machine will run faster. By doing this, the machine will run more effectively. By doing this, the machine will something something something." They are focusing on machines. But in fact we need to focus on humans, on how humans care about doing programming or operating the application of the machines. We are the masters. They are the slaves. - matz

Data types

"hello" aka String

Strings are used for storing and manipulating text in Ruby. Strings are written between quotation marks.

Making a String


name = "Robert"

# is equivalent to:

name = String.new("Robert")

Working with Strings


name = "Robert"

name.uppercase
# => "ROBERT"

"Hello #{name}"
# => "Hello Robert"

"Hello " + name
# => "Hello Robert"

'Hello #{name}'
# => "Hello \#{name}" <- Be carefull here!

Double vs. Single quotes

Use 'single quotes' only when you know that the string will not be changed.

String vs. Symbol

Symbols are frozen Strings. In Ruby the #frozen method is used to mark objects to prevent their future modification. 


name = "Robert"
name + " Miller"
# => "Robert Miller"

name = :robert
name + " Miller"
# => NoMethodError: undefined method `+' for :robert:Symbol

Numbers aka Fixnum

[] aka Array.new

An array is a Ruby data type that holds an ordered collection of values, which can be any type of object including other arrays.

Defining empty Array


names = []

# or

names = Array.new

Adding items to Array


names = [ "Mike", "John", "Steven" ]
# => ["Mike", "John", "Steven"]

fruits = Array.new(3, "apple")
# => ["apple", "apple", "apple"]

fruits << "cherry"
# => ["apple", "apple", "apple", "cherry"]

Iterating Arrays


names = [ "Mike", "John", "Steven" ]

names[1]
# "John"

names.each do |name|
  puts "Hello #{name}!"
end

# Hello Mike!
# Hello John!
# Hello Steven!

Manipulating Array


names = [ "Mike", "John", "Steven" ]

names.map { |n| n+"san" }

# => ["Mike-san", "John-san", "Steven-san"]

Searching Array


names = [ "Mike", "John", "Steven" ]

names.find { |name| name == 'Steven' }
# => "Steven"

names.reject { |name| name == 'Steven' }
# => ["Mike", "John"]

Deleting items from Array


names = [ "Mike", "John", "Steven" ]

names.delete "John"
# => "John"

names
# => ["Mike", "Steven"]

Hash

Hashes are collections of key-value pairs. Like arrays, they have values associated with indices, but in the case of hashes, the indices are called "keys." Keys can be anything that's hashable, such as integers, strings, or symbols, but they must be unique for the hash they belong. The values to which keys refer can be any Ruby object.

Creating a new Hash


drinks = { mike: 'beer', steven: 'vodka', robert: 'watter' }
# => {:mike=>"beer", :steven=>"vodka", :robert=>"watter"}

drinks[:peter]
# => nil

drinks = Hash.new("vodka")
# => {}
drinks[:peter]
# => "vodka"

Iterating through Hash


drinks = { mike: 'beer', steven: 'vodka', robert: 'watter' }

drinks.each do |name, drink|
  puts "#{name} has #{drink}"
end

# mike has beer
# steven has vodka
# robert has watter

Note about coding style


drinks = { mike: 'beer', steven: 'vodka', robert: 'watter' }

drinks = {
  mike: 'beer',
  steven: 'vodka',
  robert: 'watter'
}

Control Flow

if

Many ways how to use 'if'


if condition
  puts 'condition is not "nil" nor "false"'
end

# is equivalent to:

puts 'condition is not "nil" nor "false"' if condition

Where not to use 'if'


# wrong!

name = nil

if !name
  name = 'Mike'
end

# correct:

name ||= 'Mike'

# => name = name || 'Mike'

unless

Unless is negation of 'if'


unless condition
  # condition is false or nil
end

# Set the name to 'Mike' only if the person is not a robot.
name = 'Mike' unless person.robot?

else/elsif

When 'if' is not enough


if condition == 1
  # do this when condition is equal 1
elseif condition == 2
  # do this when condition is equal 2
else
  # otherwise do this.
end

Beware!

If you find yourself using more then one elsif in your code, you should think about refactoring it or using the case statement instead. 


case condition
  when 1 then # do something when condition is equal 1
  when 2 then # do something when condition is equal 2
  else
    # otherwise do this.
end
